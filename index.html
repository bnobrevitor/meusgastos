Configurar IA local (opcional)

Sem IA local ligada, as respostas são calculadas por regras a partir dos seus dados (sem custo, funciona para perguntas diretas). Com o Ollama rodando no seu computador (grátis, ollama.com), o agente passa a responder perguntas abertas com um modelo de verdade — tudo roda na sua máquina, nada é enviado para fora. No celular, use na URL o endereço Tailscale do computador (ex.: http://SEU-PC.ts.net:11434) em vez de localhost — assim funciona de qualquer lugar, sem expor o Ollama na internet pública.

Ajustar saldo pra bater com o banco

Corrige o saldo inicial da conta escolhida pra que o saldo calculado pelo app feche com o valor que o banco informou na última sincronização. Marque uma ou mais contas do banco pra somar (ex.: conta corrente + cartão) contra uma única conta do app. Não altera nenhum lançamento já importado.
Contas e cartões

Cartão de crédito: as despesas deixam o saldo negativo (fatura). Para pagar a fatura, faça uma transferência da conta para o cartão. Os lançamentos importados do banco caem na conta que você escolher na revisão.

Backup dos dados

Seus dados são sincronizados na nuvem (seção abaixo). Se precisar, restaure um backup antigo em JSON ou exporte seus lançamentos.

Sincronizar entre aparelhos (nuvem, grátis)

1) Crie um token grátis clicando aqui (marque só o escopo "gist") e cole acima. 2) Clique em "Criar sincronização nova" — vai aparecer um ID de sincronização: copie e cole nesse mesmo campo no seu celular, junto com um token (pode ser o mesmo), e clique em "Conectar".

Conexão bancária em tempo real (Open Finance)

Isso conecta com o Worker do Cloudflare que você publicou — ele busca os dados no Meu Pluggy. Nenhuma credencial bancária passa por aqui, só o endereço do Worker e o token combinado entre os dois. Depois de configurar, use o botão "Atualizar do banco" na aba Lançamentos.
